# wi-parser
Реализует скачивание изображений с фотоловушек, размещённых на сайте **Wildlife Insights**.
## Структура приложения
`dataset/` - содержит ПРИМЕРЫ файлов *images.csv* из различных датасетов с WI, которые содержат ссылки на скачивание изображений.
> [!IMPORTANT]
> Для корректной работы приложения в эту папку необходимо поместить **свои собственные** файлы, которые вы можете скачать, зарегистрировавшись на WI
> и запросив нужный вам bundle через их форму. 

`wi_tools/` - содержит инструменты для работы с WI, а именно: 
- `loader.py` - модуль предобработки таблиц с данными из *images.csv*.
- `parser.py` - парсер для поиска и скачивания фото с WI по набору ссылок из датафрейма.
- `sampler.py` - модуль, реализующий метод *Temperature Sampling* для обеспечения справедливой выборки данных из датасета.
- `splitter.py` - модуль с функцией разделения датасета, сформированного на базе *images.csv*, на несколько равных частей.
При этом фотографии, сделанные с одной фотоловушки, попадают строго в один и тот же сегмент (для этого в коде реализован жадный алгоритм).
- `organizer.py` - модуль пост-обработки скачанного набора фотографий, который осуществляет категоризацию изображений по месту съемки и их переименование в более читабельный формат.

`negatives.py, positives.py` - скрипты, которые посредством рассмотренных выше инструментов выполняют задачу загрузки позитивных/негативных сэмплов
для датасета, используемого нашей моделью детекции снежного барса **WildLifeCV**.
## Развертывание и установка зависимостей

> [!NOTE]
> Предполагается, что у вас уже установлен пакетный менеджер `uv`, а также система контроля версий `git`.

1. Клонируем репозиторий
```shell
git clone https://github.com/Kakoy-Nikakoy-Corp/wi-parser.git
cd wi-parser
```
2. Устанавливаем зависимости и браузер для библиотеки *playwright*
```shell
uv sync
playwright install
```
3. Создаём в корневой директории файл `.env` со следующим содержимым:
```env
WI_EMAIL="<ваш email от Wildlife Insights>"
WI_PASSWORD="<ваш пароль от Wildlife Insights>"
PART=1 # Часть датасета, которую предстоит скачать
SIGNATURE="81e3050b406016b3a2cca8b99bdaad5a" # Сигнатура для проверки корректности разбиения датасета. спрашивать у Максима :)
```
## Запуск приложения
```shell
uv run negatives.py  # Или любой другой файл
```
Необработанные изображения будут доступны в папке `download/`, а итоговый датасет - в папке `output/`. 
Кроме того, внутри датасета будет создан файл `kept_images.csv` со списком фотографий, которые программе удалось скачать (*как показывает практика, потери могут составлять до 50%
от изначального списка*).
