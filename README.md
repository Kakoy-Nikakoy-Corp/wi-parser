# 📸 WI-Parser

**WI-Parser** — это утилита для автоматизированного скачивания, обработки и организации изображений с фотоловушек, размещенных на платформе [Wildlife Insights](https://www.wildlifeinsights.org/).

Инструмент разработан для исследователей и разработчиков компьютерного зрения (в частности, для проекта **WildLifeCV** по детекции снежного барса) и позволяет эффективно формировать датасеты из сырых данных WI.

## ✨ Возможности

- 📥 **Автоматическая загрузка**: Парсинг и скачивание изображений по ссылкам из экспортированных CSV-файлов Wildlife Insights.
- ⚖️ **Сбалансированная выборка**: Реализация метода *Temperature Sampling* для справедливой репрезентации данных в датасете.
- ✂️ **Умное разделение**: Разбиение датасета на части с учетом локации фотоловушек (все фото с одной камеры попадают в один сегмент).
- 🗂️ **Организация данных**: Автоматическая категоризация скачанных фото по местам съемки и переименование файлов в читаемый формат.
- 🛠️ **Гибкость**: Модульная архитектура позволяет использовать отдельные компоненты (загрузчик, семплер, сплиттер) независимо.

## 📂 Структура проекта

~~~text
wi-parser/
├── dataset/            # Папка для ваших CSV-файлов (images.csv) от WI
├── wi_tools/           # Основные модули библиотеки
│   ├── loader.py       # Предобработка таблиц данных
│   ├── parser.py       # Логика скачивания изображений
│   ├── sampler.py      # Temperature Sampling для балансировки
│   ├── splitter.py     # Разделение датасета по камерам
│   └── organizer.py    # Пост-обработка и сортировка файлов
├── negatives.py        # Скрипт для загрузки негативных сэмплов
├── positives.py        # Скрипт для загрузки позитивных сэмплов
├── download/           # (Создается автоматически) Сырые скачанные изображения
├── output/             # (Создается автоматически) Итоговый организованный датасет
└── .env                # Файл с конфигурацией (не храните в git!)
~~~

> [!IMPORTANT]
> Папка `dataset/` содержит только **примеры** структуры файлов. Для работы вам необходимо скачать собственные файлы `images.csv` с платформы Wildlife Insights после запроса нужного bundle с данными.

## 🚀 Установка и настройка

### Предварительные требования

- Установленный [Python](https://www.python.org/)
- Пакетный менеджер [uv](https://github.com/astral-sh/uv) (рекомендуется для быстрой установки зависимостей)
- Git

### Пошаговая инструкция

1. **Клонируйте репозиторий:**
   ~~~bash
   git clone https://github.com/Kakoy-Nikakoy-Corp/wi-parser.git
   cd wi-parser
   ~~~

2. **Установите зависимости и браузер Playwright:**
   ~~~bash
   uv sync
   uv run playwright install
   ~~~

3. **Настройте переменные окружения:**
   Создайте файл `.env` в корневой директории проекта и заполните его следующими данными:

   ~~~env
   # Ваши учетные данные от Wildlife Insights
   WI_EMAIL="your_email@example.com"
   WI_PASSWORD="your_secure_password"

   # Номер части датасета для скачивания (если используется разбиение)
   PART=1

   # Сигнатура для проверки целостности разбиения (уточняйте у администратора проекта)
   SIGNATURE="81e3050b406016b3a2cca8b99bdaad5a"
   ~~~

## 💻 Использование

Для запуска процесса скачивания и обработки используйте готовые скрипты.

### Скачивание негативных сэмплов
~~~bash
uv run negatives.py
~~~

### Скачивание позитивных сэмплов
~~~bash
uv run positives.py
~~~

### Результат работы
После завершения скрипта:
1. **`download/`** — содержит все необработанные изображения.
2. **`output/`** — содержит финальный датасет с правильной структурой папок и именами файлов.
3. **`kept_images.csv`** — создается внутри папки `output/`. Содержит список успешно скачанных фотографий.

> [!NOTE]
> Из-за особенностей сети или доступности файлов на стороне WI, потери при скачивании могут составлять до **50%** от исходного списка. Всегда сверяйтесь с `kept_images.csv`.

## 🧩 Использование модулей (для разработчиков)

Вы можете импортировать инструменты `wi_tools` в свои собственные скрипты:

~~~python
from wi_tools import load_dataset, temperature_sampling

CAT_CONFIG = {
    'A': [A, 2500, 0.3],
    'B': [B, 1500, 0.4],
    'C': [C, 500, 0.5],
    'D': [D, 1000, 1],
}

# Пример загрузки данных
df = load_dataset("./dataset/my_images.csv")

# Пример выборки
sampled_df = temperature_sampling(df, 'common_name', CAT_CONFIG)

# Дальнейшая логика...
~~~
